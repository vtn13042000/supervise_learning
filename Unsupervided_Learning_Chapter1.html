Unsupervise Learning - KMean Clustering¶

Learn how to discover the underlying groups (or "clusters") in a dataset. By the end of this chapter, you'll be clustering companies using their stock market prices, and distinguishing different species by clustering their measurements.

Overview¶

“Clustering” is the process of grouping similar entities together. The goal of this unsupervised machine learning technique is to find similarities in the data point and group similar data points together.

There are many algorithms developed to implement this technique but but the most popular and widely used algorithms in machine learning are these two.

    1. K-mean Clustering
    1. Hierarchical Clustering

This chapter will focus on K-mean algorithm

This is how the algorithm works

    1. It starts with K as the input which is how many clusters you want to find. Place K centroids in random locations in your space. Now, using the euclidean distance between data points and centroids, assign each data point to the cluster which is close to it.
    1. Recalculate the cluster centers as a mean of data points assigned to it.
    1. Repeat 2 and 3 until no further changes occur.

So, how to decide which number is the best fit for K parameter for our sample?

  • One of the methods is called “Elbow” method can be used to decide an optimal number of clusters. Here you would run K-mean clustering on a range of K values and plot the “percentage of variance explained” on the Y-axis and “K” on X-axis.

  • In the picture below, you would notice that as we add more clusters after 3 it doesn't give much better modeling on the data. The first cluster adds much information, but at some point, the marginal gain will start dropping.

How many clusters of grain?¶

Since it's unable to plot 8D points on a graph to guess the number of cluster, we will use "Elbow" method as mention above.

In [ ]:
ks = range(1, 6)
inertias = []

for k in ks:
    # Create a KMeans instance with k clusters: model
    model = KMeans(n_clusters = k)
    
    # Fit model to samples
    model.fit(samples)
    
    # Append the inertia to the list of inertias
    inertias.append(model.inertia_)
    
# Plot ks vs inertias
plt.plot(ks, inertias, '-o')
plt.xlabel('number of clusters, k')
plt.ylabel('inertia')
plt.xticks(ks)
plt.show()

Then 3 is the optimal value of k
